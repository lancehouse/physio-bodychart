# PhysioBodyChart — GTK4/C App

See `../CLAUDE.md` for the full two-app architecture and shared file format.

## Build

```
ninja -C build
```

Reconfigure after adding source files: `meson setup --reconfigure build`

## Source layout

| File | Purpose |
|---|---|
| `src/canvas.c/.h` | Drawing areas, gesture input, Cairo draw pipeline |
| `src/obj_chart.c/.h` | Objective mode: zone/point types, render, sidebar |
| `src/window.c` | Sidebar UI, mode switching, dialogs |
| `src/persistence.c` | JSON save/load (json-c) — reads/writes `_session.json` |
| `src/input.c` | Stroke input logic (pressure, stylus, touch) |
| `src/session.c` | PNG/PDF export |
| `meson.build` | Source list — add new `.c` files here |

## Mode architecture

Three modes in `AppMode`: `APP_MODE_SUBJECTIVE`, `APP_MODE_OBJECTIVE`, `APP_MODE_REPORT`.

- **Sx mode** — strokes (symptoms by type), arrows, notes
- **Obj mode** — freehand zones (Allodynia/Hyperalgesia/Erythema/Temp Cool/Warm) + PPT/TS point markers
- **Rpt mode** — placeholder (report is generated by the TUI, not the GTK app)

Sidebar switches content via `g_sidebar_content_stack` (`"sx"` / `"obj"` / `"rpt"`).

## Session JSON ownership

This app writes `_session.json` only. Never touch `_assessment.json` or `_objective.json` —
those are owned by the TUI. Shared schema changes require a version bump in both apps.

Key fields written on every significant change (stroke committed, overlay changed, region changed):

```json
{
  "schema_version": 1,
  "session_name": "...",
  "patient_id": "...",
  "session_label": "...",
  "regions": ["lumbar"],
  "subjective": { "strokes": [], "notes": [], "arrows": [] },
  "objective":  { "zones": [],  "points": [] }
}
```
