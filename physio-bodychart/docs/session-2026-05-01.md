### 3. Arrow tool

New `TOOL_ARROW` mode. Workflow:

1. Press the Arrow button in the toolbar (below Note)
2. Press and drag to draw — the arrow follows the drawn path
3. Release to commit; a thin black arrow with a small filled arrowhead is placed at the body position

**Curved arrows:** A quadratic bezier is fit to the drawn path. The control point `Q` is derived from the sampled midpoint `M` at `t=0.5`:

```
Q = 2·M − 0.5·(P0 + P2)
```

Path points are tracked in a self-halving 32-point buffer during the gesture.

**Arrowhead:** Oriented along the tangent at `t=1` of the bezier (`normalize(P2 − Q)`). Head length = `min(6.0, len×0.22)` body units; half-width = `head_len×0.38`. The shaft terminates at the head base (converted from quadratic to two cubic segments).

**Deletion:** Tap/click the arrowhead to delete the arrow. Hit radius = 14 screen pixels. This check runs before palm rejection so it works even on touch.

**Undo:** Arrows are on the shared undo stack alongside strokes. An `undo_type_stack[64]` array tracks type (0 = stroke, 1 = arrow); `canvas_undo` consults it to pop the right list.

Structs added to `canvas.h`:
```c
typedef struct {
    int    view;
    double x1, y1;
    double cx, cy;   /* quadratic bezier control point */
    double x2, y2;
} ArrowAnnotation;
```
AppState fields: `arrows[MAX_ARROWS=30]`, `arrow_count`, `arrow_drawing`, `arrow_draw_view`, `arrow_x1/y1/x2/y2`, `arrow_track_x/y[32]`, `arrow_track_n`, `undo_type_stack[64]`, `undo_type_top`.

Files changed: `src/input.h` (TOOL_ARROW enum), `src/canvas.h`, `src/canvas.c`, `src/input.c`, `src/window.c`.

## Batch B — Six fixes and additions

### 1. SVG peripheral nerve overlays

`anterior peripheral n.svg` and `posterior peripheral n.svg` placed in `views/`. The overlay system now loads and renders these via `overlay_svg_draw_periph()`, using the same `push_group`/`pop_group`/`paint_with_alpha` wrapper as dermatomes.

New in `src/overlay_svg.c`:
- `g_periph[2]` array of `RsvgHandle*`
- `PERIPH_NAMES[2]` path name table
- `load_svgs_from_dir()` (renamed from `try_dir`) loads both derm + periph per directory
- `render_svg()` helper
- `overlay_svg_draw_periph()` function

New declaration added to `src/overlay_svg.h`.

---

### 2. Link summary — visible and draggable across all panels

Previously the link summary box was only rendered and draggable on the panel whose view matched `link_summary_view`. Both restrictions were removed:

- `link_summary_hit_screen()`: view check removed — the box is grabbable from any panel
- `canvas_render_view()` / `on_col_draw`: link summary rendered unconditionally

Result: the summary appears simultaneously in all visible panels and can be grabbed from any of them.

---

### 3. Curvy arrows with quadratic bezier fit

See Batch A, item 3 above. The bezier fit uses the path-tracking buffer built up during the draw gesture. Straight arrows (short or near-straight paths) produce a straight bezier automatically because `Q` collapses toward the midpoint.

---

### 4. Arrow deletion

See Batch A, item 3. Tap/click within 14 screen pixels of the arrowhead (x2, y2) to delete.

---

### 5. Longer note abbreviations

The note wizard now produces four-character abbreviations:

| Field | Old | New |
|-------|-----|-----|
| Temporal — Constant | `C` | `Con` |
| Temporal — Intermittent | `I` | `Int` |
| Depth — Superficial | `S` | `Sup` |
| Depth — Deep | `D` | `Dep` |
| Quality | `Pn`, `Ac` … | `Pain`, `Ache` … (4 chars) |

Example label: `(1)Con Sup Pain 3/7`

Updated in `src/window.c`: `wizard_commit` (temporal/depth strings) and `QUALITY_SHORT[14]`.

---

### 6. P+N note quality option

`NOTE_QUALITY_COUNT` increased from 13 → **14**. New quality added: `"P+N"` (Pins and Needles alongside Numbness). The wizard quality grid now shows 14 options: 12 in a 4×3 grid, row 3 = Other (2 cols) + P+N (2 cols).

`QUALITY_SHORT` in `window.c`:
```c
"Pain","Ache","Numb","Shrp","Dull","Hot","Cold","Itch","Craw","Elec","Shot","Buzz","Othr","P+N"
```

---

### 7. Single tick per tap

`input_motion` now returns immediately if the active stroke type is `SYMPTOM_TICK`, preventing any additional points being added after the first. Result: tapping places exactly one checkmark; dragging does not produce a chain.
