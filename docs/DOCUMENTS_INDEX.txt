📌 KEY DECISIONS YOU MADE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. File-Based Sync (Not RPC/Sockets)
   ✓ Keeps apps independent (no direct communication)
   ✓ Works across processes without extra infrastructure
   ✓ Simpler to debug and test

2. Polling (Not inotify/watchfiles)
   ✓ Zero external dependencies
   ✓ Works on any filesystem (networked or not)
   ✓ 1-second latency acceptable for clinical workflow
   ✓ Can upgrade to inotify later (1-line code change)

3. Atomic Writes (temp file + rename)
   ✓ Prevents corruption if process crashes mid-write
   ✓ Reader never sees partial file
   ✓ Matches industry best practice

4. Debouncing with Timestamps
   ✓ Prevents feedback loops
   ✓ App ignores its own writes within 2 seconds
   ✓ Minimal overhead, no complex state machine

5. Clean Ownership Model
   ✓ GTK owns: subjective + objective + ui sections
   ✓ Python owns: assessment section
   ✓ No overlapping writes = no conflicts
